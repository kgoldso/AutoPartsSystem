# AutoParts Order System — Прогресс разработки (v2.0)

Система управления продажей автозапчастей с ролевой моделью доступа (RBAC).

---

## 👥 Акторы и Роли (согласно Use Case)
1. **Гость** — анонимный поиск и просмотр каталога.
2. **Клиент** — регистрация заказов, отслеживание статуса и истории.
3. **Менеджер** — оформление заказов от имени клиента, управление статусами.
4. **Администратор** — управление пользователями, каталогом и аналитика (отчеты).
5. **Кладовщик** — резервирование, отгрузка и инвентаризация остатков.

## Этап 2 — Бизнес-логика (Service Layer) 🔄 В РАБОТЕ
**Цель:** Реализация функциональных связей из UML-диаграммы.

- [ ] **Core / Identity:**
    - Реализация логики «Обеспечить вход пользователя» (Authentication)
    - Проверка прав доступа (Role-based Authorization)
- [ ] **OrderService (Клиент/Менеджер):**
    - `PlaceOrder` — создание заказа с автоматическим `ReserveParts`
    - `GetOrderHistory` — история заказов (фильтр по ClientID или общий для Менеджера)
- [ ] **WarehouseService (Кладовщик):**
    - `RegisterShipment` — регистрация отгрузки (смена статуса + списание со склада)
    - `UpdateInventory` — обновление остатков на складе
    - `GetPendingShipments` — список заказов, ожидающих отгрузки
- [ ] **AdminService (Администратор):**
    - `GenerateSalesReport` — отчет по продажам
    - `GenerateStockReport` — отчет по остаткам склада
    - `ManageUsers` — управление аккаунтами (Клиент/Менеджер/Кладовщик)
